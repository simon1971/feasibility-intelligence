# Narrow B Execution Plan

## Executive summary
Path B should proceed, but only in a narrow, disciplined form.

The goal is **not** to build a broad feasibility engine now.
The goal is to build a **higher-confidence rapid screening workflow** for:
- Sunshine Coast LGA only
- residential subdivision only
- a tightly defined set of verification tasks

The commercial question is:

> Can a strong candidate move from low-confidence `Proceed with caution` to medium-confidence `Proceed with caution` quickly and repeatably?

If yes, the MVP becomes materially more valuable.
If no, it remains mainly a triage tool.

---

## Strategic objective
Improve the planning verification layer, not the breadth of the product.

This means:
- no broader geography
- no broader use cases
- no new UI work as a priority
- no parser expansion yet
- no volume scaling yet

---

## Scope definition
### Included
- Sunshine Coast LGA only
- Residential subdivision screening only
- Top-tier candidate verification workflow
- Confidence uplift testing
- Operator-led evidence capture

### Excluded
- Other councils
- Other development types
- Full automation of planning interpretation
- Client-facing product polish
- Large-batch site screening expansion

---

## Core hypothesis
If the verification step can be tightened, then:
- strong candidates can be upgraded with more confidence
- screening output becomes more commercially useful
- the MVP moves from "triage only" toward "rapid confidence-weighted screen"

---

## Success criteria
Path B is working if the following become true:
1. top-tier candidates can be reassessed with more verified planning fields
2. confidence can move from **low** to **medium** on at least some strong sites
3. verification time is materially lower and more repeatable
4. recommendation logic stays disciplined while becoming more useful

---

## Primary workstreams
### Workstream 1 — verification throughput
Improve speed and repeatability of checking:
- zone
- local plan / precinct
- minimum lot size / frontage
- flood
- steep land / landslide
- biodiversity / waterways
- prior application history

### Workstream 2 — verified-field capture discipline
Use the verified-field operator worksheet consistently so every top-tier site is checked the same way.

### Workstream 3 — confidence uplift testing
Track whether stronger candidates can move from:
- low-confidence `Proceed with caution`

to:
- medium-confidence `Proceed with caution`

### Workstream 4 — repeatability
Test whether a second batch of strong candidates behaves similarly under the improved workflow.

---

## Immediate priority sites
Use these as the primary test set:
1. 39 Park Rd, Nambour
2. 16 Mons Road, Buderim
3. 18A Quambi Place, Buderim

Reason:
- they are currently the strongest candidates
- they represent different site archetypes
- if confidence cannot be improved on these, broader scaling is hard to justify

---

## Tactical sequence
### Step 1
Use the verified-field operator worksheet on the top 3 candidates.

### Step 2
Improve source-gathering procedure for authoritative council checks.

### Step 3
Measure what remains unresolved after the improved workflow.

### Step 4
Reassess recommendation + confidence after verification pass.

### Step 5
Decide whether confidence uplift is real enough to justify continued Path B investment.

---

## Decision gates
### Gate A — continue Path B
Continue if:
- at least one top-tier site reaches medium confidence
- verification workflow becomes more repeatable
- recommendation outputs become more commercially persuasive

### Gate B — pause / revert to triage model
Pause or narrow further if:
- top-tier sites remain low confidence despite structured verification
- critical fields remain too expensive or too slow to verify
- confidence uplift is negligible

---

## What not to do next
Do not:
- broaden to other councils
- add more UI/product polish first
- automate parser work first
- keep screening large batches of middling sites

These would increase motion without addressing the bottleneck.

---

## Recommended next move
The immediate next operational task under Narrow B is:

> improve and test the planning verification workflow on the top 3 candidates until it is clear whether confidence can be meaningfully lifted.

That is the decisive test for whether Path B is commercially worth pursuing further.
