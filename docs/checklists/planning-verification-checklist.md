# Planning Verification Checklist

## Purpose
This checklist sits between:
- initial listing review
- and final Phase 1 recommendation

Its purpose is to convert a promising listing-led screen into a more decision-useful, evidence-based site screen.

This checklist exists because the first five real manual screens showed the main bottleneck was **authoritative planning verification**, not candidate sourcing.

---

## Use standard
For every Phase 1 site:
- do **not** treat listing claims as verified planning facts
- complete this checklist before upgrading confidence
- record source references for every verified item
- if an item cannot be verified, mark it `unknown`

Status values:
- `verified`
- `unknown`
- `not applicable`
- `constraint identified`

## Decision discipline notes
### Proceed
Use only when:
- major planning controls are verified
- no material overlay blocker is evident
- unknown count is low

### Proceed with caution
Use when:
- the site is still promising
- but one or more meaningful constraints remain unresolved
- enough evidence exists to justify deeper work

### Hold / verify
Use when:
- too many critical items remain unknown
- or a core planning field is still unverified

### Do not proceed
Use when:
- a verified screen-level blocker exists
- or the site is clearly outside the MVP use case
