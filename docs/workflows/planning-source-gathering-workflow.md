# Planning Source-Gathering Workflow

## Purpose
This workflow is the operational bridge between:
- a promising listing-led screen
- and a more evidence-backed Phase 1 recommendation

It exists because the first five real manual screens showed the dominant bottleneck was not sourcing sites or ranking them.

The bottleneck was gathering **authoritative planning inputs quickly enough**.

---

## When to use this workflow
Use this workflow when a site is:
- a real candidate
- within Sunshine Coast LGA
- relevant to the residential subdivision MVP
- promising enough that a listing-only review is no longer sufficient

Typical trigger:
- a site has reached `Proceed with caution`
- or a strong `Hold / verify` outcome that needs a verification pass

---

## Output required
By the end of this workflow, the operator should have recorded:
1. verified zone
2. verified local plan / precinct (if applicable)
3. verified minimum lot size / subdivision trigger
4. verified frontage / access notes
5. verified flood status
6. verified steep land / landslide status
7. verified biodiversity / waterways status
8. verified prior application history
9. source references for all of the above

If any of these remain unresolved, they must stay explicitly marked as `unknown`.

---

## Step 1 — Prepare the site record
Before opening planning sources, collect:
- full address
- suburb
- listing URL
- gross site area from listing
- any obvious listing claims that need verification

Examples of claims to verify:
- “medium density zoning”
- “development potential”
- “duplex/townhouse potential”
- “approved plans”
- “future scope”

Important:
- listing claims are leads, not evidence

---

## Step 2 — Open the authoritative source set
Use the council/planning source set in this order.

### Source A — Sunshine Coast Development.i / site report path
Use this first for site-level planning context.

Goal:
- confirm site identity
- confirm zone
- identify local plan / precinct relevance
- identify overlays where available
- identify application history entry points

Capture:
- zone
- local plan / precinct
- flood indication if shown
- any direct application links or history references

### Source B — Sunshine Coast planning scheme mapping / local plan maps
Use this when Development.i/site report does not provide enough precision.

Goal:
- verify local plan area
- verify precinct / sub-precinct
- confirm mapped planning context

Capture:
- plan name
- precinct name
- map layer reference if available

### Source C — Flood / overlay mapping
Use this to check:
- flood
- steep land / landslide
- biodiversity / waterways
- any other environmental overlay material to subdivision

Capture each overlay separately.

### Source D — Application history / PD-style record
Use this to check:
- prior approvals
- refusals
- lodged applications
- patterns that materially support or weaken the site

Capture:
- application number(s)
- application type
- status
- why relevant

---

## Step 3 — Verification sequence
Complete the checks in this order.

### 3.1 Zone
Question:
- What is the current authoritative zone for this site?

Record:
- zone name
- source
- whether the listing claim matched reality

### 3.2 Local plan / precinct
Question:
- Does a local plan apply, and if so which precinct matters?

Record:
- plan name
- precinct name
- source

### 3.3 Minimum lot size / subdivision trigger
Question:
- What control actually governs subdivision feasibility at screen level?

Record:
- minimum lot size
- frontage rule if any
- source
- whether still uncertain

### 3.4 Frontage / access
Question:
- Is access practically workable at screen level?

Record:
- corner block / single frontage / dual access / unknown
- source
- notes on limitations

### 3.5 Flood
Question:
- Does the site appear affected by flood mapping in a way material to screening?

Record:
- pass / caution / fail / unknown
- source

### 3.6 Steep land / landslide
Question:
- Is steep land or landslide mapping likely to materially constrain outcome?

Record:
- pass / caution / fail / unknown
- source

### 3.7 Biodiversity / waterways
Question:
- Are there mapped environmental constraints that could materially affect subdivision potential?

Record:
- pass / caution / fail / unknown
- source

### 3.8 Prior application history
Question:
- Has this site or a closely related proposal already been tested through council processes?

Record:
- relevant application(s)
- support / neutral / caution effect
- source

---

## Step 4 — Evidence discipline
For each field, record:
- value
- status
- source
- short note

Do not record:
- vague impressions as facts
- unverified agent language as planning truth
- assumed lot-size rules without reference

Use phrases such as:
- `Verified from council source`
- `Listing claim only — not yet verified`
- `Overlay not confirmed in this pass`
- `Application history not located yet`

---

## Step 5 — Decision conversion
After source gathering, reassess:
- recommendation
- confidence
- critical unknown count

### Typical expectation
If the source-gathering workflow succeeds, a strong site may move from:
- `Proceed with caution / Low confidence`

to:
- `Proceed with caution / Medium confidence`

In some cases it may move to:
- `Proceed`

Or fall back to:
- `Hold / verify`
- `Do not proceed`

if verified constraints are found.

---

## Step 6 — Operator completion standard
A verification pass is complete when:
- all decisive planning fields have been checked
- unresolved items are explicitly marked unknown
- source references are recorded
- recommendation is updated accordingly

A verification pass is not complete when:
- listing language is copied into the assessment without verification
- overlays are assumed absent because they were not checked
- application history is skipped on a promising site

---

## Recommended operator cadence
For top-tier candidates:
1. do listing-led screen
2. do this source-gathering workflow immediately after
3. then update the assessment recommendation

Do not leave a promising site sitting in low-confidence status if the source-gathering step has not yet been attempted.

---

## Practical use note
This workflow is now the highest-leverage operational step in the MVP.

The model can already rank sites reasonably.
The next commercial improvement comes from getting verified planning inputs faster and more consistently.
