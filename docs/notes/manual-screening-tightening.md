# Manual Workflow Tightening Notes

## Placeholder handling
- Keep `verified_reference_needed` and `threshold_is_placeholder` visible in every assessment.
- Use the phrase `Placeholder pending verified planning reference` consistently.
- Do not let a placeholder-derived figure produce high confidence.

## Report wording
Use conservative phrasing:
- `screen-level indication`
- `requires planning scheme confirmation`
- `not yet verified from authoritative source`
- `suitable for deeper feasibility review, not determination`

Avoid:
- `permitted`
- `complies`
- `can be subdivided`
when the underlying source chain is still incomplete.

## Recommendation consistency
- `Proceed` only when all material gates are addressed and no major constraint remains untested.
- `Proceed with caution` when a pathway appears plausible but one or more constraints could materially change outcome.
- `Hold / verify` when evidence is incomplete.
- `Do not proceed` only on a verified screen-level blocker.

## Parser timing
Defer parser automation until:
- 5 genuine manual screens are completed
- recommendation consistency is stable
- average assessment time is acceptable
