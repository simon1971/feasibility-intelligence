This matrix compares the first five real screened sites side by side so the Phase 1 MVP can be judged on patterns rather than isolated screens.

## Pattern summary
### 1. The model is ranking sites in a believable order
The ordering broadly matches expected intuition:
- strongest raw strategic parcel at the top
- strongest explicit redevelopment signal near the top
- weaker or more generic suburban opportunities lower down
- lifestyle/control case at the bottom

### 2. Most sites are clustering in Hold / verify
This is the most important operational finding.

The clustering is not because the model cannot distinguish quality.
It is because too many decisive verification fields remain unresolved.

### 3. Listing-led optimism is not enough
The model is not being fooled by:
- generic “future scope” language
- larger-than-average lot size alone
- agent redevelopment phrasing without verified planning support

That is positive.

---

## What the matrix reveals about the MVP
### Working well
- relative ranking
- conservative recommendation discipline
- separation of stronger and weaker signals
- rejection of unverified listing hype as proof

### Not working well enough yet
- movement from candidate identification to planning verification
- confidence uplift after initial screen
- efficient conversion of promising sites into evidence-backed recommendations

---

## Practical implication
The MVP is currently strongest as a:
- **prioritisation and triage engine**

It is not yet strongest as a:
- **high-confidence rapid planning verdict engine**

That distinction should shape next development choices.

---

## Recommended next actions
### 1. Use the planning verification checklist against these five sites
The new checklist should now be applied to the same five sites to see whether stronger evidence materially shifts recommendations.

### 2. Track which fields most often block confidence
Likely recurring blockers:
- zone confirmation
- local plan / precinct confirmation
- minimum lot size confirmation
- flood check
- biodiversity / waterways check
- prior application history

### 3. Decide whether recommendation logic needs slight tuning
Possible adjustment:
- allow a site with strong raw characteristics and no negative signals to sit more comfortably in `Proceed with caution`
- but only if planning unknowns are important rather than fatal

### 4. Build a repeatable review cadence
After every batch of 5 screened sites:
- compare outcomes
- compare blockers
- refine workflow, not just score weights

---

## Current conclusion
The first five real screens support continuing the MVP.

But they also show that the next improvement should be:
- **verification workflow quality**

not:
- UI expansion
- parser automation
- broader geographic scope
