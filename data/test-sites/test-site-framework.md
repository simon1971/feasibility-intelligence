# 20 Test-Site Framework

## Purpose
Build a deliberate test set that pressures the MVP without broadening scope.

## Target mix
Create 20 test sites across these buckets:

1. **Clear likely proceed** — 4 sites
   - Larger regular lots
   - Apparent residential zoning
   - Limited obvious physical constraints

2. **Proceed with caution** — 6 sites
   - Moderate constraint complexity
   - Some overlay uncertainty
   - Access/frontage questions

3. **Hold / verify** — 6 sites
   - Missing threshold clarity
   - Precinct uncertainty
   - Unclear application history

4. **Do not proceed** — 4 sites
   - Outside LGA, non-target use case, or obvious severe constraints

## For each test site capture
- site_id
- suburb
- archetype
- why included
- expected recommendation
- expected confidence
- major unknowns

## Test objective
The framework should prove:
- consistency of analyst decisions
- usefulness of the 12-rule screen
- whether the report output is commercially credible
- where verified planning reference gaps are slowing throughput

## Important
Do not treat test-site expectations as truth. They are calibration hypotheses only.
